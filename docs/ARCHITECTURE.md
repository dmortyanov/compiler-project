# Архитектура MiniCompiler

MiniCompiler реализует классический многопроходный пайплайн компиляции.

## Архитектура
1. **Лексический анализ** (`src/lexer/`)
   Преобразует исходный код в поток токенов. Поддерживает ключевые слова, идентификаторы и литералы.
2. **Синтаксический анализ** (`src/parser/`)
   Строит абстрактное синтаксическое дерево (AST). Использует рекурсивный спуск.
3. **Семантический анализ** (`src/semantic/`)
   Проверяет типы, область видимости, декларирует функции (в т.ч. extern) и массивы. 
4. **Генерация промежуточного представления (IR)** (`src/ir/`)
   AST обходится паттерном Visitor. Генерируется линейный трёхадресный код. Поддерживаются базовые блоки, CFG (Control Flow Graph) и PHI-функции (в форме параметров).
5. **Оптимизация IR** (`src/ir/optimizer.cpp`, `optimization_passes.cpp`)
   Конвейер локальных и межблочных оптимизаций: инлайнинг (Inlining), Copy Propagation, CSE (Common Subexpression Elimination), свёртка констант, удаление мертвого кода (DCE). Работает итеративно до стабилизации графа.
6. **Генерация кода (x86-64 NASM)** (`src/codegen/`)
   - Распределение регистров: стековое или алгоритмом Linear Scan Register Allocation (LSRA).
   - Вызовы функций соблюдают System V AMD64 ABI.
   - Массивы размещаются в локальном стековом фрейме функции, выделение через вычисление смещений.
